# Кареты HTML (inctructive)

Статический веб-проект: заставка, переходы между страницами, персонажи с выбором цвета команды, колоды на третьей странице. Сборщика нет — достаточно браузера; для удобной разработки добавлен локальный сервер с автообновлением.

## Требования

- **Node.js** (LTS) — только если хотите запускать режим разработки с перезагрузкой страницы при сохранении файлов.

## Быстрый старт (разработка)

1. Откройте терминал в папке `inctructive` (там лежит `package.json`).

2. Установите зависимости (один раз):

   ```bash
   npm install
   ```

3. Запустите сервер с live-reload:

   ```bash
   npm run dev
   ```

   Откроется браузер с адресом вроде `http://127.0.0.1:5500/index.html`. При изменении HTML, CSS или JS в этой папке страница обновится сама.

4. Остановка: в терминале нажмите `Ctrl+C`.

### Windows (PowerShell)

Если путь с кириллицей, надёжнее так:

```powershell
Set-Location -LiteralPath "путь\к\inctructive"
npm install
npm run dev
```

## Запуск без Node.js

Можно открыть `index.html` двойным щелчком или через «Открыть файл» в браузере (`file://`). Часть сценариев (особенно с `fetch` к локальным файлам) может вести себя иначе, чем через `npm run dev` — для проверки лучше использовать локальный сервер.

## Страницы

| Файл | Назначение |
|------|------------|
| `index.html` | Стартовая заставка и загрузка |
| `second-page.html` | Следующий экран сценария |
| `third-page.html` | Экран с колодами и слотами |

Стили и скрипты подключаются из той же папки (`style.css`, `characters.css`, `animation.css`, `*.js` и т.д.).

## Данные в браузере

Настройки персонажей и команд могут сохраняться в **localStorage** (ключ вроде `teamsData`). Чтобы сбросить состояние, очистите данные сайта для этого адреса в настройках браузера.

## Структура (кратко)

- `img/` — изображения
- `data.json` — данные для скриптов (если используется)
- `package.json` — скрипт `dev` и зависимость `live-server`
